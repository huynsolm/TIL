### Context
n8n을 사용하여 첫 번째 워크플로우 자동화를 구현했습니다. 특정 시나리오로, 디스코드 메시지를 수신하면 해당 메시지가 트리거로 작용하여 에이전트가 실행됩니다. 이 에이전트는 메시지를 TIL(오늘 배운 것) 형식으로 변환하고, 해당 콘텐츠를 GitHub에 push합니다. push가 완료되면 README.md 파일에 해당 인덱스가 추가되고, 모든 작업이 끝나면 완료 메시지가 발신됩니다.

### Core
* **디스코드 메시지 수신 트리거**: 디스코드에서 메시지를 수신하여 트리거를 활성화합니다.
* **GitHub에 코드 push**: 변환된 코드를 n8n을 통해 GitHub에 push합니다.
* **README.md 업데이트**: 최신 인덱스를 README.md 파일에 추가합니다.
* **완료 메시지 발신**: 모든 작업이 완료되면 디스코드로 완료 메시지를 발신합니다.

```n8n
# 예시 노드 설정
- Start [Discord Trigger]
- Code Conversion [JavaScript]
- GitHub Push [n8n GitHub Node]
- README Update [Webhook]
- Send Completion Message [Discord Node]
```

### Insight
n8n으로 워크플로우를 처음 구성해보았는데, 어렵지 않다는 것을 깨달았습니다. 이해해야 할 것은 각 노드의 input과 output입니다. 이를 명확히 이해하면 워크플로우 구성은 쉬워집니다. 이 과정을 도와준 후츠릿에게 감사합니다.
