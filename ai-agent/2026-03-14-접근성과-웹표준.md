### Context

접근성과 웹표준을 강화하기 위해 키보드 흐름과 시맨틱 구조를 점검하는 것이 중요합니다. 접근성 표준인 WCAG 2.4.1은 사용자가 웹 콘텐츠를 순차적으로 탐색할 때 주요 콘텐츠에 직접 접근할 수 있도록 합니다.

### Core

HTML을 사용하여 테이블을 구성할 때, 목적을 명확히 하는 방식으로 접근성을 향상시킬 수 있습니다. 예를 들어, 다음은 주간 주문 현황을 나타내는 코드입니다:

```html
<table aria-label="주간 주문 현황">
  <thead>
    <tr><th>...</th></tr>
  </thead>
  <tbody>
    <tr><td>...</td></tr>
  </tbody>
</table>
```

CSS를 이용하여 스크롤 마진을 설정하면 사용자가 건너뛰기 링크를 활용할 때 도착 위치를 조정할 수 있습니다:

```css
main {
  scroll-margin-top: 5rem;
}
```

### Insight

표준은 호환성 비용을 줄이며, 웹에서는 시맨틱 마크업이 유지보수와 보조기기 해석의 안정성을 증가시킵니다. 반복 블록의 건너뛰기 링크 제공 여부를 확인하여 사용자의 접근성을 높이는 것이 중요합니다. 또한, 큰 테이블이나 리스트의 초기 렌더링 범위를 줄여 LCP 변화를 모니터링하는 것도 성능 점검에 필수적입니다.

자주 사용하는 페이지 1개를 선택하여 Tab 키만으로 탐색을 시도하고 접근성에 걸림돌이 되는 지점을 기록하세요.
