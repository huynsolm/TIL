### Context
웹 페이지 성능과 유지보수성을 개선하기 위해 CSS 최적화와 접근성 기준을 준수하는 방법을 정리합니다. 중복 CSS 스타일을 줄이고, 접근성을 높여 웹 페이지의 전반적인 성능과 사용성을 향상시키고자 합니다.

### Core
* **CSS 최적화**: 중복 스타일을 유틸리티 클래스로 묶고 `:where()` 선택자를 사용하여 특이성을 낮출 수 있습니다.
  ```css
  .btn, .btn-primary, .card :where(h2, p) {
      margin: 0;
  }
  ```
  `:where()`는 특이성이 항상 0으로, 유지보수에 매우 유리합니다.

* **접근성(대비비) 체크**: 본문 텍스트 대비비를 4.5:1 이상으로 설정합니다(WCAG 2.1 준수).
  - Chrome DevTools를 사용해 `Elements` → `Accessibility`/`Color Picker`에서 대상 대비비를 확인하고, 기준 미달 색상을 수정합니다.

* **성능 개선**: 이미지를 비동기로 로딩하고(`loading="lazy"`), `srcset`과 `sizes`를 적용하여 초기 렌더링 부하를 줄입니다.

* **성능 측정**: 주요 성능 지표로 LCP(≤2.5s)와 CLS(≤0.1)를 측정합니다. Lighthouse 또는 PageSpeed를 사용하여 배포 전후 성능을 비교 기록합니다.

### Insight
CSS 특이성을 낮추는 최적화와 함께, 접근성 기준을 준수함으로써 웹 페이지의 유지보수와 일관성을 높일 수 있습니다. 특히, 메인 페이지에서 대비비 기준을 넘지 못했던 텍스트를 수정하고 LCP를 측정한 결과, 성능이 크게 향상되었습니다. 이러한 방법들은 웹 개발자들에게 실질적인 성능 개선을 제공할 수 있습니다.
